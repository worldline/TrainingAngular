Main Concepts Forms - Custom Validators

  • In previous samples, we used the required validator with our FormControl
  • To specify that every field is required, we will use a Validator. A validator returns a map of errors or null if it detects no error.
  • The native validators provided by Angular are:
    • required : Validators.required to ensure that a control is not empty
    • minlength : Validators.minLength(n) to ensure that the value entered has at least ncharacters
    • maxlength : Validators.maxLength(n) to ensure that the value entered has at most ncharacters
    • pattern: Validators.pattern(p) to ensure that the value matches the regular expression p
  • In a code-driven form, you can apply several validators at once, by using an array, on a FormControl on a FormGroup. constructor(fb: FormBuilder){ this.myFormGroup = fb.group({ 'categoryControl': fb.control('', [Validators.required, Validators.minLength(3)]), 'detailsControl': fb.control('', Validators.required) }); }
  • In a template-driven form, adding a required field in a template-driven form is also really straightforward. you just have to add the required attribute to the inputs. required is a provided directive, and will automatically add the validator to this field. Same thing with minlength and maxlength. < input name="details" [(ngModel)]="model.details" required minlength="3" >
