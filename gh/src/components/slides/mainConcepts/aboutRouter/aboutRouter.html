Main Concepts Router - Introduction

  • Usually, navigate in an application means reload a new page based on a new url
  • But remember that we are in a Single Page Application
  • So the router will take care of this behavior
  • The router has its own API, it's not mandatory to use it
  • And we didn't use it in previous samples
  • It's a little bit the same idea than in AngularJS : views are mapped to url
  • But with an important difference: the granularity of views, views can contains other views...
