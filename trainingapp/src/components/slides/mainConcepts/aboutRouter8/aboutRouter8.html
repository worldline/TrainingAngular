Main Concepts Router - limit access to routes

Finally, we can now create routes, sub routes and load them dynamically. We're missing one last feature, the control to leave or access a component. The basic example is to authorize the access to a route only for a logged user which has enough rights. But it could also be don't authorize the user to leave this form component while he didn't fill the form...

  • For that, with Angular we are using Guards
  • We have guards for the following scenarios (copy/paste from official doc)
    • CanActivate: to mediate navigation to a route
    • CanActivateChild: to mediate navigation to a child route
    • CanDeactivate: to mediate navigation away from the current route
    • Resolve: to perform route data retrieval before route activation
    • CanLoad: to mediate navigation to a feature module loaded asynchronously
  • In our previous example, let's add a checkbox to ask the user to accept some conditions before being able to load the todo module
