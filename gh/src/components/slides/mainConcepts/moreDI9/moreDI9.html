Main Concepts Dependency Injection - DI Exceptions

As we told before, in order to inject a service into an other entity ( component / service / pipe ), Angular has to know which types are required by this entity.

And we also told that, to do that, Angular looks for metadatada associated to the given entity. Below is a transpiled component Cmp1 which depends on the service Srv1

var Cmp1 = (function () { function Cmp1( srv1 ) { // Transpiled methods and attributes are there // srv1 has to be injected // ... // ... } Cmp1 = __decorate([ core_1.Component({ selector: 'cmp1', template '<p>I am CMP1 </p>' }), // These metadata tells angular which type has // to be injected __metadata('design:paramtypes', [srv1_1.Srv1]) ], Cmp1); return Cmp1; })();
