Main Concepts Dependency Injection - DI Exceptions

  • typescript will not generate these metadata for standard classes (typescript is not Angular, and DI is based on Angular)
  • While working with components, the @Component annotation let typescript know that it has to generate these metadata so services are injected autommatically
  • But services which depend on other services doesn't use any annotation by default
  • Add @Inject on the constructor parameters: constructor(@Inject(MyService) myService: MyService){}
  • Or @Injectable on the service class: @Injectable class MyService{ constructor(dep: MyDepService){ //... } }
